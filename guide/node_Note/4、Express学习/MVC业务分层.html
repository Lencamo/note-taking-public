# 一、设计模式：

# 1、MVC 架构

  • M（Model）：模型（domain、service、mapper 等等）
      —— 封装和处理数据
  • V（View）：视图（html、ftl、jsp 等等）
      —— 展示数据
  • C（Controller）：控制器（Controller、Servlet 等等）
      —— 处理请求（如：接收参数、调用 service 层代码、控制页面跳转等等）

# 2、MVVM 架构

  • M（Model）：
      —— 准备数据
  • V（View）：
      —— 展示页面
  • VM（ViewModel）：
      —— 视图和模型（视图和数据的转换）

# 一、MVC 设计模式

Controller（控制层类似于一个中间媒介的存在）

  接触的 MVC 模式应用场景：

# 1、Javaweb 中

【 Javabean：pojo / entity 】

其中：

  • 表现层：Controller（web 文件夹）、View
  • 业务逻辑层（业务层）：Model（service 文件夹）
      —— 它通常还会配备一个 serviceImpl 文件夹
  • 数据访问层（持久层）：Model（dao / mapper 文件夹）
  • 数据传输层*：Model（dto 文件夹）

# 2、SSM 中

SSM 即 Spring 、SpringMVC、Mybatis 框架

其中：

  • 表现层 ——> SpringMVC / Struts2
  • 业务逻辑层 ——> 手写需求
  • 数据访问层 ——> MyBatis / Hibernate

【 Spring 则控制整个项目中对象的创建与管理：IoC（控制反转）和 AOP（面向切面编程）】

重点：

SSM 框架的整合 (opens new window) 🤔

const UserController = {
  // 异步💖处理（send的时机）
  addUser: async (req, res) => {
    const { username, password, age, phone } = req.body

    // 分层✨二
    await UserService.addUser

    res.send({
      ok: 1
    })
  }

  updateUser: async (req, res) => {
    const { username, password, age, phone } = req.body
    const _id = req.params.id

    await UserService.updateUser(_id,username, password, age, phone)

    res.send({
      ok: 1
    })
  }
}

module.exports = UserController
