# web 开发模式

目前主流的 web 开发模式有两种：

  • 基于服务端渲染的传统 web 开发模式
       —— (SSR：后端组装页面)

  • 基于前后端分离的新型 web 开发模式
       —— (BSR：前端组装页面)

使用选择：

  可以从 SEO 、耗时、开发效率、服务器资源等角度选择。

① 服务端渲染 SEO 强

企业级网站

② Vue，React 等前端框架的 SSR (server side render)技术能够很好的解决 SEO 问题！

后台管理

  当然，一些网站为了同时兼顾首页的渲染速度和前后端分离的开发效率，采用了首屏服务器端渲染+其他页面前后端分离的开发模式。

# 一、前后端分离（BSR）

  前后端分离是一种架构模式，说通俗点就是后端项目里面看不到页面（JSP|HTML）,后端给前端提供接口，前端调用后端提供的 REST 风格接口 就行，前端专注写页面（html | jsp）和渲染（JS | CSS| 各种前端框架）；后端专注写代码就行。前后端分离的核心：后台提供数据，前端负责显示

  目前前后端分离主要有两种方式：

① node.js 作为中间层进行全栈式的开发

② 前后端通过约定的数据接口进行通信，以相同的数据格式如 JSON 格式进行数据传输
