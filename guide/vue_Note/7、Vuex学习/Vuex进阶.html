# Vuex：

官方介绍：

Vuex (opens new window) 可以帮助我们管理共享状态（数据），并附带了更多的概念和框架。

# 一、项目结构

  使用 Vuex 需要遵守的规则：

  • 应用层级的状态应该集中到单个 store 对象中
  • 提交 mutation 是更改状态的唯一方法
  • 异步逻辑都应该封装到 action 里面

  对应大小项目，如果 store 文件太大，可以将将 action、mutation 和 getter 分割到单独的文件。项目结构如下：

├── index.html
├── main.js
├── api
│   └── ... # 抽取出API请求
├── components
│   ├── App.vue
│   └── ...
└── store
    ├── index.js          # 我们组装模块并导出 store 的地方
    ├── actions.js        # 根级别的 action
    ├── mutations.js      # 根级别的 mutation
    └── modules
        ├── cart.js       # 购物车模块
        └── products.js   # 产品模块

# 二、插件

  Vuex 的 store 接受 plugins 选项，这个选项暴露出每次 mutation 的钩子。Vuex 插件就是一个函数，它接收 store 作为唯一参数。

# 1、简单示例

  • router / index.js
import Vue from 'vue'
import Vuex from 'vuex'

Vue.use(Vuex)

// 2、插件功能🤔
const myPlugin = (store) => {
  // 当 store 初始化后调用
  store.subscribe((mutation, state) => {
    // 每次 mutation 之后调用
    // mutation 的格式为 { type, payload }
  })
}

const store = new Vuex.Store({
  // ...
  // 1、插件使用
  plugins: [myPlugin]
})

export default store

# 2、应用：Vuex 数据持久化存储 ✨

# ① 方式一：vuex 原生写法

  每当我们更新 store 数据时，手动将持久化存储的数据存储到 sessionStorage 中。

const myPlugin = (store) => {
  // 当 store 初始化后调用
  store.subscribe((mutation, state) => {
    // 每次 mutation 之后调用
    // mutation 的格式为 { type, payload }
    sessionStorage.setItem('vuex', JSON.stringify(state)) //因为vuex都是通过mutation来修改数据的，所以在每次修改的时候都把state存到本地内存中
  })
}

# ② 方式二：第三方包方式

  利用vuex-persistedstate (opens new window)插件自动实现持久化存储。

vuex 的插件包叫做vuex-persistedstate@3.2.1版本(配合 vue2 使用, 默认最新版是配合 vue3 使用)

下载：

yarn add vuex-persistedstate@3.2.1
1

简单示例：

  • store / index.js
import Vue from 'vue'
import Vuex from 'vuex'
// 1、导入包
import createPersistedState from 'vuex-persistedstate'

Vue.use(Vuex)

const store = new Vuex.Store({
  // ...
  // 2、使用插件
  plugins: [createPersistedState()]
})

export default store

插件配置（下面演示的全部为默认值：可省略 👀）：

/* vuex数据持久化配置 */
plugins: [
  createPersistedState({
    // 1、存储方式：localStorage、sessionStorage、cookies
    storage: window.localStorage,
    // 2、存储的 key 的key值
    key: 'vuex',
    path: []
    // 3、指定需要持久化存储state
    render(state) {
      return { ...state } // 默认存储了state中所有的数据（es6扩展运算符）
    }
  })
]

import { cartStore } from './userStore'

const dataState = createPersistedState({
  // 用于部分持久化状态的任何路径的数组
  // paths: ['carMsg'],
  paths: [...],

  reducer(state, paths) {
    return {
      // 只储存state中的user
      user: state.userInfo //只存储userInfo信息并且在localStorage中userInfo命名替换成user
    }
  }
})

const store = new Vuex.Store({
  state: {
    userInfo: {}
  },
  // 模块化管理
  modules: {
    cartStore
  },
  // 插件管理
  plugins: [dataState]
})

export default store

# 2、拓展：vue 数据持久化存储

  • store.js
export default new Vuex.Store({
  // Vuex中的数据
  state: {
    token: sessionStorage.getItem('newToken'),
    userInfo: JSON.parse(sessionStorage.getItem('newUserInfo') || '[]')
  },
  mutations: {
    // 登录成功后的token值
    setToken(state, newToken) {
      state.token = newToken
      sessionStorage.setItem('newToken', newToken)
    },
    // 用于存储获取的用户信息
    setUserInfo(state, newUserInfo) {
      state.userInfo = newUserInfo
      sessionStorage.setItem('newUserInfo', JSON.stringify(newUserInfo))
    }
  }
})
