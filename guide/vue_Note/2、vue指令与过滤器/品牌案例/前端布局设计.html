# RWD

官方简介：

响应式 Web 设计（Responsive Web Design），一种可以自动适应不同的访问设备(显示器、笔记本电脑、平板电脑或手机)的新型网站设计模式

现代布局方式

例如多栏布局、伸缩盒和网格默认是响应式的。

# 一、传统布局（静态布局）

# 1、简单介绍

传统的布局模式有以下四种：

  • 文档布局

      块（Block）：用于网页中的部分（节）
      行内（Inline）：用于文本

  • 浮动布局（float）

      用于形成 inline-block 元素，容易引起父类高度塌陷

  • 定位布局（position）

      用于元素的明确位置，解决的往往都是小的、元素层面的问题

# 2、总结

  布局的传统解决方案，基于盒状模型，依赖 display 属性 + position 属性 + float 属性。它对于那些特殊布局非常不方便，比如，垂直居中就不容易实现。

# 二、流式布局(百分比布局)

auto、100%、max-width、min-width 等等

# 三、rem 和 viewpoint

rem 布局重某种角度来讲也是属于弹性布局。

  • 固定的 px 改为会随着父元素大小而改变的 em
  • 随着根元素的 font-size 而改变的 rem

# 1、em 使用

1em / 25% 大小相对于父元素的 font-size 大小

  浏览器默认的字体大小为 16px，若父元素没有设置 font-size 大小的话，那么1em = 16px。

# 2、rem 使用

1rem 相对于 html 元素的 font-size 大小

公式：

动态的 font-size 基准值 = 屏幕动态宽度 / @no

  • 屏幕宽度：由@media 动态设置
  • @no：为同一规定的屏幕划分份数
html {
  /* 用html根元素设置的font-size值会覆盖浏览器默认的font-size值 */
  font-size: 12px;
}

使用（设计稿一般是二倍图）：

  • font-size 基准值 = 当前设计稿大小(750px / 2) / @no
  • @no：自定义 或 默认都统一为 10 吧

① 半自动转换：

# 3、vw / vh 使用

1vw 相对于 当前视口的 font-size 大小

公式：

1vw = 视口宽度 / 100

使用（以 iPhone6/7/8 为例）：

  • 视口宽度：UI 设计图为 2 倍图（2x 模式），即视口宽度为 750px / 2 = 375px。
  • vw 大小：1vw = 375px / 100 = 3.75px

① 半自动转换：

  • vscode 插件：
    • px2vw： 设置视口宽度（插件默认为 375px）

配置：

② 全自动转换

  • postcss-px-to-viewport (opens new window)：自动转换

配置：

  • postcss.config.js
module.exports = {
  plugins: {
    '# postcss-px-to-viewport-8-plugin': {
      unitToConvert: 'px', // 要转化的单位
      viewportWidth: 375, // UI设计稿的宽度
      unitPrecision: 6, // 转换后的精度，即小数点位数
      propList: ['*'], // 指定转换的css属性的单位，*代表全部css属性的单位都进行转换
      viewportUnit: 'vw', // 指定需要转换成的视窗单位，默认vw
      fontViewportUnit: 'vw', // 指定字体需要转换成的视窗单位，默认vw
      selectorBlackList: ['wrap'], // 指定不转换为视窗单位的类名，
      minPixelValue: 1, // 默认值1，小于或等于1px则不进行转换
      mediaQuery: true, // 是否在媒体查询的css代码中也进行转换，默认false
      replace: true, // 是否转换后直接更换属性值
      exclude: [/node_modules/], // 设置忽略文件，用正则做目录名匹配
      landscape: false // 是否处理横屏情况
    }
  }
}

# ① flex-direction

  定义轴方向

  • column : 从上到下堆叠
  • column-reverse : 从下到上堆叠
  • row : 从左到右堆叠（默认值）
  • column-reverse : 从右到左堆叠

# ② flex-wrap

  • wrap : 必要时换行
  • nowrap : 不换行，子元素 width 可能会被挤压（默认值）

# ③ flex-flow :

  相当于 flex-direction + flex-wrap 属性

# ④ justify-content

  多余空间子元素整体水平对齐分配

  • center : 子元素整体水平中心对齐
  • flex-start : 子元素整体向左对齐（默认值）
  • flex-end : 子元素整体向右对齐
  • space-between : 子元素的水平行间分配 ✨
  • space-around : 子元素的水平行前、间、后分配 ✨
  • space-evenly : 子元素的水平行前、间、后平均分配（了解）

# ⑤ align-items

  多余空间子元素整体垂直对齐分配

  • center : 子元素整体垂直中心对齐
  • flex-start : 子元素整体向上对齐
  • flex-end : 子元素整体向下对齐
  • stretch : 子元素上下拉伸填充（默认值）
  • baseline : 按各子元素的文字基线对齐（了解）

使用将 justify-content 和 align-items 属性设置为居中，然后 flex 项目将完美居中

# ⑥ align-content

  针对有多行子元素的多余空间分配，必须和 flex-wrap：wrap 配合使用

  • center : 子元素整体垂直中心对齐
  • flex-start : 子元素整体向上对齐
  • flex-end : 子元素整体向下对齐
  • space-between : 子元素的垂直行间分配 ✨
  • space-around : 子元素的垂直行前、间、后分配 ✨✨
  • stretch : 子元素等量上下拉伸填充（默认值）

# ② flex 🍗

  flex 属性相当于flex-grow + flex-shrink + flex-basis（在瓜分、弥补父项的剩余、不足空间时，各子项获得、失去的弹性放大比、弹性缩小比、项目占据的主轴空间大小值）

  其中：flex-basis 属性的默认值为 auto，即项目的本来大小。当设置为一个固定值时，此属性可以让项目将占据固定空间

几种简写形式：

  • flex: initial （默认值）   等效于 flex: 0 1 auto（不放大会缩小）
  • flex: auto   等效于 flex: 1 1 auto（放大且缩小）
  • flex: none   等效于 flex: 0 0 auto（不放大也不缩小）

使用方式： 🚩

.item {
  /* 后面两个属性是可选的 */
  flex: < 'flex-grow' > [< 'flex-shrink' > || < 'flex-basis' >];
}

  例如：flex: 1; 是 flex: 1 1 auto 的简写。而 flex 属性的默认值是 flex: 0 1 auto
