# this 指向

  this 的指向是当我们调用函数的时候确定的，调用方式的不同决定了 this 的指向不同。

调用方式 this 指向 描述
普通函数 window
构造函数 实例对象 (原型对象里的方法 指向的是 实例对象)
对象方法 该方法所属对象
事件绑定方法 绑定事件对象
定时器函数 window
立即执行函数 window

（一般指向的是 🚩 调用者）

// 1、普通函数
function fn() {
  console.log(this) // Window { …… }
}
// window.fn()
fn()

// 2、构造函数
function Star(name, age) {
  this.name = name
  this.age = age

  console.log(this) // Star { name: '鹿先生',age: 22}
}
var that
Star.prototype.sing = function () {
  that = this
  console.log(this) // Star { name: '鹿先生',age: 22}
}
var lencamo = new Star('鹿先生', 22)
lencamo.sing()

console.log(lencamo) // Star { name: '鹿先生',age: 22}
console.log(that === lencamo) // true

// 3、对象方法
var obj = {
  say: function () {
    console.log(this) // {say: f()} 其实就是 obj
  }
}
obj.say()

// 4、事件绑定方法
var btn = document.querySelector('button')
btn.onclick = function () {
  console.log(this) // 打印了button整个元素
}

// 5、定时器函数
// window.setTimeout(function () {
setTimeout(function () {
  console.log(this) // Window { …… } 🤔
}, 3000)

// 6、立即执行函数
// (function () {
//   console.log(this)
// })()

# 一、call()方法

function.call(thisArg, arg1, arg2, ...)
1

描述：

  使用一个指定的 this 值和单独给出的一个或多个参数来调用一个函数。简单的来说，即：

  • 可以调用函数
  • 可以改变函数的 this 指向

# 1、作用

  • 调用函数
function fn() {
  console.log('函数被调用了')
  console.log(this) // Window { …… }
}

# 二、apply()方法

function.apply(thisArg, [argsArray])
1

描述：

  这个函数的语法与 call() 几乎相同，但根本区别在于，call() 接受一个参数列表，而 apply() 接受一个参数的单数组。

# 1、与内置函数搭配

  第二个参数是函数接受的参数，以数组的形式传入，且当第一个参数为 null、undefined 的时候，默认指向 window(在浏览器中)

// 找出数组中最大/小的数字
const numbers = [5, 6, 2, 3, 7]

// 使用 Math.min/Math.max 以及 apply 函数时的代码
let max = Math.max.apply(null, numbers) // 基本等同于 Math.max(numbers[0], ...) 或 Math.max(5, 6, ..)
let min = Math.min.apply(Math, numbers)

# 三、bind() 方法

function.bind(thisArg[, arg1[, arg2[, ...]]])
1

描述：

返回一个新的函数 ，在 bind() 被调用时，这个新函数的 this 被指定为 bind() 的第一个参数，而其余参数将作为新函数的参数，供调用时使用。简单的来说，即：

  • 会返回一个原函数的拷贝，并拥有指定的 this 值和初始参数
  • 不会调用原函数
  • 可以改变原函数内部的 this 指向

# 执行特点：

  bind 是返回绑定 this 之后的函数，便于稍后调用；apply 、call 则是立即执行。（所以，bind 常和定时器搭配使用）

即：在不调用函数的情况下，改变 this 指向

# 传参比较：

  关于传参：call 是参数列表，而 apply 是数组，且 apply 和 call 是一次性传入参数，而 bind 可以分为多次传入

var arr = [1, 10, 5, 8, 12]
var max = Math.max.bind(null, arr[0], arr[1], arr[2], arr[3])
console.log(max(arr[4])) //12，分两次传参

# 四、总结 ✨

# 1、个人描述

  一般情况下，是不存在 this 指向问题，因为一般情况下 this 指向的是调用者。

  但也有其他特殊的情况，比如和下面的几个函数搭配使用的时候，就会存在 this 指向问题：构造函数、对象方法、事件绑定方法、定时器函数、立即执行函数等等

② 简介

  • 在普通函数中，this 总是指向调用它的对象，如果用作构造函数，this 指向创建的对象实例。
  • 箭头函数在全局作用域声明，所以它捕获全局作用域中的 this，this 指向 window 对象。

# 4、箭头函数与普通函数的区别

  • 外形不同
  • 箭头函数都是匿名函数
  • 箭头函数不能用于构造函数，不能使用 new
  • 箭头函数中 this 的指向不同
  • 箭头函数不绑定 arguments，取而代之用 rest 参数 … 解决
// 1、普通函数
function A(a) {
  console.log(arguments)
}
A(1, 2, 3, 4, 5, 8) //  [1, 2, 3, 4, 5, 8, callee: ƒ, Symbol(Symbol.iterator): ƒ]

// 2、箭头函数
let C = (...c) => {
  console.log(c)
}
C(3, 82, 32, 11323) // [3, 82, 32, 11323]
