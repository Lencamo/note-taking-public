# 请求限制：

  出于安全性考虑，小程序官方对数据接口请求做了以下限制：

  • 只能请求 HTTPS 类型的接口
  • 必须将接口的域名添加到信任列表中

关于合法性域名配置自行解决即可

# 一、小程序中的数据请求

# 1、注意事项

  小程序的宿主环境不是浏览器，是微信客户端。所以小程序是不存在跨域问题的。

  Ajax 技术的核心是依赖于浏览器中的 XMLHttpRequest 对象。所以小程序中的数据请求是不能叫做“Ajax 请求”的，应该叫做“网络数据请求”。

# 2、合法性校验

  在开发阶段，为了我们方便，可以在开发环境开启不校验请求域名、HTTPS 证书等选项。

  • project.config.json
{
  // 编译相关配置
  "setting": {
    "urlCheck": false // 关闭合法域名校验
  }
}

# 3、网络请求

  对于网络请求，我们可以在微信官方文档的 API 部分找到 wx.request()方法。

官方文档：网络请求 (opens new window)

① 请求示例

Page({
  data: {
    articleList: []
  },

  onLoad: function (options) {
    this.getArtclList()
  },

  getArtclList(){
    wx.request({
      url:'http://www.note-book.cn/api/post',
      method: 'POST',
      data: {
        name: '中国',
        numb: '22'
      }
      success: (res) => {
        // console.log(res)
        this.setData({
        articleList: res.data
        })
      }
    })
  }
})

② 常配置项

  • 常用属性
属性 默认值 说明
url 开发者服务器接口地址
method GET HTTP 请求方法
header application/json 设置请求的 header
data string/object/ArrayBuffer 请求的参数

只有 url 属性是必填项

  • 回调函数
属性 说明
success 接口调用成功的回调函数
fail 接口调用失败的回调函数
complete 接口调用结束的回调函数（调用成功、失败都会执行）
  • 了解
属性 默认值 说明
dataType json 返回的数据格式
responseType text 响应的数据类型

# 二、API Promise 优化

# 1、原生的 API 问题

  默认情况下，小程序官方提供的异步 API 都是基于回调函数实现的，例如上面的网络请求 API：

wx.request({
  method: '',
  url: '',
  data: {},
  success: () => {}, // 请求成功的回调函数
  fail: () => {}, // 请求失败的回调函数
  complete: () => {} // 请求完成的回调函数
})
