# 一、接口的 REST 风格

简介：

REST 风格，不是行为规范，只是推荐的一种约定方式。

优点：

  • 书写简化
  • 隐藏了资源访问行为

注意事项：

  描述模块的名称通常使用复数，表示一类资源。例如：users、books ……

# 1、传统方式

  回忆前面 MongoDB 学习部分，写的接口：

请求类型 描述
POST(增) /api/user/add 或者 /api/addUser
GET (删) /api/user/delectable/4 或者 /api/delectUser/4
POST (改) /api/user/update/4 或者 /api/updateUser/4
GET (查) /api/user/list?page=1&limit=10 或者 /api/getUser

简单的说 🚩：REST 风格就是通过 请求类型 来隐藏 访问行为

# 2、REST 风格方式 🍗

请求类型 描述
POST(增) /api/users
DELETE (删) /api/users/4
PUT (改) /api/users/4
GET (查) /api/users

# 3、使用

import { getCommentListAPI } from '@/api'

export default {
  methods: {
    async getCommentListFn() {
      const { data: res } = await getCommentListAPI({
        id: this.$route.query.id,
        limit: 15,
        offset: (this.page - 1) * 15
      })
      // console.log(res)
      res.commentList = res.comments.forEach((obj) => this.commentList.push(obj))
    }
  }
}
