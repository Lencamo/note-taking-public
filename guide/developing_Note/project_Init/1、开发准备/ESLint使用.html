# 一、ESLint 简介

# 1、rules 说明
"off" 或 0 关闭规则
"warn" 或 1 开启规则 使用警告级别的错误：warn (不会导致程序退出)
"error" 或 2 开启规则 使用错误级别的错误：error (当被触发的时候，程序会退出)

  ESLint 虽然可以定义很多的 rules，以及通过 extends 来引入更多的规则，但是说到底只是检查 JS 语法。

问题解决：后面的 plugins 解决

# 2、extends 配置（规范）

直接用

  • eslint:all：ESLint 内置的所有规则
  • eslint:recommended：ESLint 内置的推荐规则

安装

  • eslint-config-airbnb-base：airbnb 的 JS 规范（airbnb-base）
  • eslint-config-standard：standard 的 JS 规范（standard）
  • eslint-config-prettier：Prettier 的 JS 规范（prettier）

# 3、plugins 配置（修复）

  如果需要检查 Vue 中的 template 或者 React 中的 jsx，需要引入插件来增强 ESLint 的检查能力和范围。

{
  "plugins": [
    "vue", // 等价于"eslint-plugin-vue"
    "@typescript-eslint" // 等价于@typescript-eslint/eslint-plugin
    "prettier" // 等价于eslint-plugin-prettier
  ]
}

# 4、pre-commit 钩子

  在提交代码的时候，自动修复格式问题，并且只修复自己改动的代码。可以通过配置 git hooks 的 pre-commit 钩子来实现这个目的

① 下载

npm i husky
npm i lint-staged -D
1
2

② 配置

  • package.json
{
  "lint-staged": {
    "src/**/*.{js,vue}": ["eslint --fix", "git add"]
  },
  "husky": {
    "hooks": {
      "pre-commit": "lint-staged"
    }
  }
}

# 5、综合写法（应用）

  后面有具体使用内容，这里只是简单的展示。

module.exports = {
  // 前面规范、后面修复
  // 官方地址：https://www.npmjs.com/package/eslint-plugin-prettier
  // eslint-plugin-prettier + eslint-config-prettier
  extends: ['standard', 'plugin:prettier/recommended'],

  // eslint + eslint-plugin-vue
  //官方地址：https://eslint.vuejs.org/user-guide/#usage
  extends: ['eslint:recommended', 'plugin:vue/recommended'] // vue2
  // extends: ['eslint:recommended', 'plugin:vue/vue3-recommended'] // vue3
}

# 2、配置

  • .eslint.js
module.exports = {
  extends: ['standard', 'plugin:prettier/recommended'],
  // extends: ['eslint:recommended', 'plugin:prettier/recommended'],

  rules: {
    'prettier/prettier': 'error',

    // 默认的规则
    'no-console': process.env.NODE_ENV === 'production' ? 'warn' : 'off',
    'no-debugger': process.env.NODE_ENV === 'production' ? 'warn' : 'off',

    // 自定义的规则
    'space-before-function-paren': ['warn', 'never'],
    'vue/multi-word-component-names': 'off' // 关闭名称校验
  }
}

# 三、vue-cli 与 ESLint

注意：

为了防止格式化时受到 Prettier 插件的影响，建议暂时关闭 Prettier 插件。

# 1、安装

  vue 官方 eslint 插件配置。

下载：

  • vue-cli 方式（推荐）
# 如果没有安装eslint
vue add @vue/eslint

# 官方地址：https://github.com/vuejs/vue-cli/tree/dev/packages/%40vue/cli-plugin-eslint
vue add @vue/cli-plugin-eslint
