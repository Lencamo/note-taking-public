# 3、promise 的状态

  Promise 对象通过自身的状态，来控制异步操作。其中，Promise 实例具有三种状态：
待定（pending） 初始状态，既没有被兑现，也没有被拒绝。
已兑现（fulfilled） 意味着操作成功完成。
已拒绝（rejected） 意味着操作失败。

  其中，这三种状态的变化途径只有两种：

  • 从未完成到成功
  • 从未完成到失败

# 4、promise 的结果

  异步操作成功后，Promise 实例传回一个值（value），状态变为 fulfilled。

  异步操作失败后，Promise 实例抛出一个错误（error），状态变为 rejected。

function ajax(url) {
  return new Promise((resolve, reject) => {
    setTimeout(() => {
      // resolve({"username":"lencamo","password": 666666}); // 成功
      reject({ code: 404, info: '页面不存在' }) // 失败
    }, 1000)
  })
}

var promObj = ajax('http://www.liulongbin.top:3006/api/getbooks')

promObj
  .then((res) => {
    console.log('ajax请求成功', res)

    console.log(promObj)
  })
  .catch((error) => {
    console.log('ajax请求失败', error)

    console.log(promObj)
  })

# 三、Promise 封装 Ajax

  前面的两个 ajax 封装案例都只是写了个空架子，下面我们来看看完整的代码。

即对原生的 XHR 方式请求服务器资源的代码进行封装优化。

function ajax(url) {
  return new Promise((resolve, reject) => {
    // 1、创建xhr对象
    var xhr = new XMLHttpRequest()

    // 2、发生get请求
    xhr.open('GET', url, true) // 采用异步✨，所以要加上true
    xhr.send()

    // 监听事件
    xhr.onreadystatechange = function () {
      if (xhr.readyState === 4) {
        // 3、返回响应数据
        if (xhr.status >= 200 && xhr.status <= 300) {
          // 成功
          resolve(JSON.parse(xhr.responseText))
        } else {
          // 失败
          reject(xhr.responseText)
        }
      }
    }
  })
}

ajax('http://www.liulongbin.top:3006/api/getbooks')
  .then((res) => {
    console.log(res)
  })
  .catch((error) => {
    console.log(error)
  })
