# 一、静态 ip 网络

# 1、节点-网络适配器

  将 Centos 7.6 的网络连接模式，选择为 桥接模式

# 2、获取一个静态 ip ✨

su root

# 随机获取一个可用的ip
dhclient
# 查看分配的ip（后面将其配置为静态ip）
ifconfig

# 3、局域网的网段说明

  通常情况下，在当前 WiFi① 环境下，分配的节点静态 ip 是在同一个网段的，即：是可以进行网络通信的。

  但如果更换了 WiFi②，那么有可能存在已经配置的好 Linux 节点和 WiFi② 不在同一个网段，从而导致无法联网。

所以，打造一个多节点的 Linux 环境是有必要的。

# 4、节点-网卡配置

  找到ifcfg-ens33文件。

cd /etc/sysconfig/network-scripts
ls
vi ifcfg-ens33
1
2
3

  配置如下：

# 1、更改部分
BOOTPROTO=static #由dhcp改为static
ONBOOT=yes #有no改为yes

# 2、新增部分
IPADDR=192.168.6.66 # 静态ip（注意：前面三个要和本地主机保持一致）
NETMASK=255.255.255.0 # 子网掩码
GATEWAY=192.168.6.1 # 网关（注意：要和本地主机的网关一致）
DNS1=119.29.29.29 # DNS服务器

# 二、动态 ip 网络

# 1、节点-网络适配器

  将 Centos 7.6 的网络连接模式，选择为 NAT 模式

# 2、节点-网卡配置

  找到ifcfg-ens33文件。

vi  /etc/sysconfig/network-scripts/ifcfg-ens33
1

  将里面的 ONBOOT=no 改为 ONBOOT=yes（让系统启动时自动激活网卡）。

# 三、window10 联网

  为 VMware 添加一个 NAT 模式的网络。

一般情况下，WMnet0 表示的是桥接模式，VMent8 表示的是 NAT 模式（并且只能有一个 NAT 模式的网络）

# 1、VMware 配置

  编辑 ——》虚拟网络配置，然后找到 VMent8 做如下配置：

① 勾选服务

  • 将主机虚拟适配器连接到此网络（勾选后才能够在主机 ipconfig 中看到 VMnet8）
  • 勾选本地 DHCP 服务（动态 ip）

② 子网配置

子网 ip

  • 主页中配置子网 ip：192.168.55.0（第三个不和本地一样即可）

NAT 设置

  • 网关 ip：192.168.55.1

DHCP 设置

  • 起始 ip 地址、结束 ip 地址（随便填写一个区间即可，只要不包含网关 ip 即可）

# 2、节点-网络适配器

  将 window 10 的网络连接模式，选择为自定义中的 VMnet8(NAT 模式)

# 3、其他

  当然，在个人本地主机上也可以将 VMware Network Adapter Vmnet8 的 IPv4 的 ip 地址自定义为一个静态的 ip。
