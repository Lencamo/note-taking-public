# 一、Sass 与 Scss

  Sass 输出样式的风格有四种风格，默认为 nested 风格。

  • nested（嵌套）
  • compact（紧凑）
  • expanded（扩展）
  • compressed（压缩）

常规的 web 前端 css 风格为expanded（扩展）风格。但使用 sass 时，为了减少文件体积，建议使用 compressed（压缩）风格

# 1、Sass

  Sass 是一种动态样式语言，由Ruby开发者设计和开发，采用缩排语法，相较于 css 多出了许多功能（如：变量、嵌套、运算、混入、继承、指令、颜色处理、函数等）。

# 2、Scss

  由于 Sass 采用的是缩排语法，对于 css 前端的 web 开发者很不友好，所以 Scss 诞生了。

sass 从第三代放弃了缩进风格，兼容了 css 代码，即 Sass 3 变成了 Scss（sassy css）

# 二、Sass 安装与配置

# 1、vscode 与 Sass

① 插件安装与配置：

  要将 sass 文件转为 css 文件，可以在 vscode 中安装Live Sass Compiler插件

注意：以前的该插件已经弃用，但原作者标明了新插件地址（貌似作者换了 🤔）

  • 配置
{
  /* 压缩设置 */
  "liveSassCompile.settings.formats": [
    // This is Default.
    {
      "format": "compressed", // nested, expanded, compact, compressed
      "extensionName": ".min.css",
      "savePath": "~/../css" // 为 null 表示当前目录
    }
  ],

  /* 排除目录 */
  "liveSassCompile.settings.excludeList": ["/**/node_modules/**", "/.vscode/**"],

  /* 是否生成对应的map */
  "liveSassCompile.settings.generateMap": false,

  /* 是否添加兼容前缀 如： -webkit- , -moz- ... */
  "liveSassCompile.settings.autoprefix": ["> 1%", "last 2 versions"]
}

# 四、拓展

# Sass、Less、Stylus 的区别

① 环境

  • Sass 需要 Ruby 环境
  • Less 需要引入 less.js
  • Stylus 需要安装 node

② 变量

/* Sass */
$color: #00c;

/* Less */
@color: #00c;

/* Stylus */
mainColor = #00c;
