# 二、三元条件函数

  三元条件函数可以对简单的@if指令进行优化。

$theme: 'dark';

.container {
  @if $theme == 'light' {
    color: #000;
  }
  @else {
    color: #FFF;
  }
}

// 等效于：

.container {
  color: if($theme == 'light', #000, #FFF)
}

# 三、@use

  从其他Sass样式表加载mixin，function和变量，并将来自多个样式表的CSS组合在一起，@use加载的样式表被称为“模块”，多次引入只包含一次（即：每一个@use相对独立）。

  总的来说，可以将@use看做的对@import的升级。

# 2、私有变量

  在上面的_global.scss和_common.scss文件中，我们发现其内部的所有变量都可以被其他文件引入。

  若要将某些变量变为私有变量，可以在变量名前加上-。

// _global.scss
$-font-size: 28px

// users.scss
@use 'users/global' as g1;

body {
  font-size: g1.$-font-size; // 会报错👀
}

# 四、@forward

  通过 @forward加载一个模块的成员，并将这些成员当作自己的成员对外暴露出去，类似于es6中的export… ，通常用于跨多个文件组织Sass库。

# 1、使用原因

  在上面的模块化中，我们发现一个问题 —— 将user.scss的任务交给 users / index.scss 后，user.scss不能直接使用uses下的样式🤔。

@use 'common';

  font-size: common.$font-size;

// 迷之操作（手动转发）

# 五、@at-root

  @at-root可以使被嵌套的选择器或属性跳出嵌套。

思考：为什么要跳出嵌套？

可以增加它的作用范围、针对于有待移动需求的元素

# 1、应用示例

  场景的应用场景就是CSS与BEM命名规范。

.block {
  width: 1000px;
  @at-root #{&}__element {
    font-size: 12px;
    @at-root #{&}--modifier {
      font-size: 16px;
    }
  }

  @at-root #{&}--modifier {
    font-size: 14px;
  }
}

# 2、限制条件

  默认@at-root只会跳出选择器嵌套，而不能跳出@media或@supports，如果要跳出这两种，则需要使用@at-root(without: media)、@at-root(without: supports)。

  其中，这种语法的关键词有四个：

  • all（表示所有）
  • rule（常规css）:默认选项
  • media
  • supports
